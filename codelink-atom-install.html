CodeLink Atom IDE

Installing Atom

Head on over to https://atom.io, and download the appropriate version for your OS. This installer should be self-extracting, and pretty straightforward.

Installing the CodeLINK Package

Once Atom is installed, head up to the "File" menu, and select "Settings". This should open a new tab with various settings modules down the left-hand column. Select the "Install" module. In the top search box, type "codelink-tools" (without quotes), and press the "Install" button.

Using Atom/CodeLINK

With both of these items installed, Atom will be looking for files ending with ".lcs" (no quotes) to determine that this file should be handled by CodeLINK. Once a file is opened/edited, CodeLINK will run a background process (CodeLink_ServiceCmd.exe) which is going to be doing the actual parsing/compilation work.

NOTE: The CodeLink package and CodeLink_ServiceCmd.exe require TCP and UDP access to communicate between Atom and CodeLink_ServiceCmd.exe. You may need to allow network access to "CodeLink_ServiceCmd.exe" for the CodeLink package to function properly!

While editing a .lcs file, CodeLINK parsing results will be displayed directly in the editor window, as well as in the "CodeLink Parser" console, at the bottom of the screen.

In the long term, this package will also facilitate creating a 'solution', a complete package of multiple projects, and compiling the lot, as well as debugging. At the moment, we're pretty much just doing the parse job, but hey, it's a start!

Updating the CodeLINK Package

Ideally, Atom would inform you of an updated package, or you could simply open the "File" -> "Settings" tab, head to "Packages" and click "Update" on the "CodeLINK" package to check for updates manually. Unfortunately, we're still working out some bugs! As such, if you need to update your Atom package, you'll need to go into "Task Manager" -> "Details", and kill the "CodeLink_ServiceCmd.exe" manually before attempting an update. Sorry for the inconvenience!
